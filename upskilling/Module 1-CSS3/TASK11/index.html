Task 11: DevTools & VS Code Debug Guide

Use this page to practise Chrome DevTools. Each panel below targets a specific debugging or testing skill. Follow the steps outlined in each card.

🔍

Inspect Elements & Styles

  1. Right-click any element on this page
  2. Select Inspect
  3. The Elements panel shows the HTML structure
  4. The Styles panel on the right shows all applied CSS rules
  5. Try toggling a property checkbox to disable a style temporarily
  6. Edit a value (e.g. change color: red) live
📱

Device Toolbar (Responsive Testing)

  1. Open DevTools (F12)
  2. Click the Device Toolbar icon (or Ctrl+Shift+M)
  3. Select a device preset: iPhone SE, iPad, Pixel 7, etc.
  4. Resize by dragging the handles or typing a custom width
  5. Observe how the navigation stacks vertically below 768px
  6. Note how font sizes and card grids adapt to screen width
📦

Box Model Inspector

  1. Inspect the .eventCard below
  2. In the Styles panel, scroll down to the Box Model diagram
  3. Hover over each section (margin, border, padding, content) to highlight it
  4. Click a value to edit it live — watch the card resize in real time
  5. Use this to debug cramped or misaligned layouts
🌐

Network Tab – Check CSS Load

  1. Open DevTools → click the Network tab
  2. Reload the page (Ctrl+R)
  3. Filter by CSS using the filter buttons
  4. Click styles.css in the list
  5. Check the Status column — it should show 200
  6. View the Headers and Preview tabs for details
🎨

Live CSS Editing in DevTools

  1. Select the #mainHeader element in the Elements panel
  2. In Styles, click on the background-color value
  3. Change it to #c0392b — the header turns red
  4. These changes are temporary — refresh resets them
  5. To keep changes, copy the CSS and paste into your styles.css in VS Code
🛠️

VS Code Tips for CSS

  1. Use Live Server extension for instant browser refresh on save
  2. Use CSS Peek to jump to a class definition from HTML
  3. Use IntelliSense for CSS autocomplete
  4. Use the built-in Problems panel (Ctrl+Shift+M) for syntax errors
  5. Format CSS with Prettier (right-click → Format Document)
  6. Use Ctrl+F to search across files for a selector name
